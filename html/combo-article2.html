    Digital Marketing code

    As a producer and distributor of alcohol products, Diageo is committed to marketing responsibly and in full compliance with the rules of our Digital Marketing code.
    Diageo web services

    The Horizon platform provides restful services (referred to as Neo Services) that are available for all brand sites and provide core capabilities like age check, user profile management, country based services etc.
    Coding standards

    We require all agencies to adhere to our coding standards to make sure any websites developed for Diageo brands reflect the highest standards.
    SEO

    SEO is the key to getting higher organic (unpaid/ natural) rankings on search engine result pages. Use SEO to increase the quality and quantity of traffic to a website, and the specific web pages that sit within it.

    The three core elements of SEO are:

    • Technical Optimisation,
    • Content Optimisation and
    • Link Building

    Read our SEO checklist
    Analytics

    Introducing Horizon Analytics, which enhances the capabilities of your Google Analytics account.

    Horizon Analytics is a minimised JavaScript library that is used to contain the Google Analytics Tracking Code (GATC). Horizon Analytics uses your existing Google Analytics account but enriches it with key performance metrics. You don’t need prior knowledge of Google Analytics or the tracking code but a sound knowledge of JavaScript and web technologies would be useful.

    Download the Horizon Analytics guide

    Accessibility

    All Diageo digital projects need to meet accessibility criteria.

    All Diageo websites should be single A compliant, or follow our Bronze guidelines by default. However, for strategic projects and strategic brands where many consumers will be visiting the site, it’s important to be AA compliant or follow at least our Silver guidelines.

    Click here to find out more

    Security

    Applications that do not adhere to industry best practices for coding are significantly more likely to be compromised, which could expose the data of Diageo, and Diageo’s customers. This can result in financial, reputational or regulatory damage to Diageo.

    Diageo has defined best practices to ensure that we develop and deploy secure web applications. These best practices include access control, authorisation, logging, session management, data protection and threat management.

    Download our security best practices

    These best practices should be closely followed by all developers and administrators developing, deploying and managing web applications for Diageo.
    Browser compliance

    Diageo wants to make its websites accessible to the largest possible audience. Browser compliance plays a big part in that.

    The world is changing and Internet Explorer is no longer the dominant browser in the world. Indeed, it was overtaken by Chrome as far back as March 2012.

    To make our websites accessible to the largest audience, we are asking our agencies to create websites that work for the following browsers:

    • Chrome
    • Firefox
    • Safari
    • Internet Explorer 11 onwards
    • Android Mobile
    • Safari Mobile

    The operating systems you have to test on are Win 7, Win 8 and 8.1 and Mac OS X.

    If you are developing a local site, please check here whether you need to support other browsers.
    Responsive / adaptive design

    Developing a responsive website is no longer a choice for agencies – it’s mandatory. Customers simply expect it, but not only that – Google will now penalise sites that are not responsive by ranking them much lower.

    Mobile phones, tablets and similar gadgets are more and more the devices of choice for people to access web content. So it's now more urgent than ever that Diageo digital projects are optimised for mobile.
